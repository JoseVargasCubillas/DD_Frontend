Movimiento como puntuación,
no como espectáculo.

Diez animaciones que le dan vida al sitio sin romper su tono editorial. Cada una con un propósito narrativo: revelar, enfocar, confirmar. Si no aporta, no se anima.

— 01

Cinco principios.

El motion del sitio se rige por cinco reglas. Si una animación rompe alguna, no se implementa.
Principios
rectores
— 01

Puntuación, no espectáculo.

Cada animación cumple un rol narrativo: revelar lo que entra, enfocar lo importante, confirmar la acción. Si no aporta lectura, se elimina.

Filosofía
rectora
— 02

Curvas humanas.

Default cubic-bezier(.2,.8,.2,1) — aceleración rápida, desaceleración suave. Imita el gesto humano de soltar un objeto sobre una mesa.

Easing
system
— 03

Tiempos contenidos.

200ms para feedback inmediato (hover), 400-800ms para reveals, 1.2-1.6s para counters narrativos. Nunca más de 2 segundos. Las prisas se notan; la lentitud también.

Duraciones
en bandas
— 04

El italic llega después.

La palabra en cursiva del titular entra con un beat adicional respecto al resto. Mismo gesto que en lectura humana: el énfasis ocurre milisegundos después del enunciado.

Detalle
tipográfico
— 05

Honra el silencio.

Toda animación respeta @media (prefers-reduced-motion: reduce). Quien necesita silencio visual lo recibe sin perder funcionalidad.

Accesibilidad
obligatoria

"Las decisiones fiscales difíciles se toman temprano, no a media noche. La estructura más elegante es la que nadie tuvo que defender de urgencia."

Haz scroll dentro de este bloque para ver cómo se condensa la barra superior.

El cambio de padding y tamaño tipográfico es sutil — entre 18px y 8px en padding, entre 18px y 13px en la marca. Suficiente para registrar que la página se desplazó sin gritar.

Es importante que el cambio ocurra en una sola transición coordinada, no en saltos. Por eso ambos elementos comparten cubic-bezier(.2,.8,.2,1) de 400ms.

Cuando regresas arriba del bloque, el nav vuelve a su estado expandido con la misma curva. La simetría temporal es parte de la sensación de "todo en su lugar".

JSfunction countUp(el, target, duration = 1600) {
  const start = performance.now();
  const ease = t => 1 - Math.pow(1 - t, 3);  // ease-out cubic
  const tick = (now) => {
    const p = Math.min(1, (now - start) / duration);
    el.textContent = Math.round(target * ease(p));
    p < 1 && requestAnimationFrame(tick);
  };
  requestAnimationFrame(tick);
}

— A11y · prefers-reduced-motion

El sistema honra el silencio.

Toda animación respeta la preferencia del sistema operativo del usuario. Si tiene "reducir movimiento" activado, todas las transiciones se vuelven instantáneas y los reveals se sustituyen por fades de 10ms. La funcionalidad no se pierde.

@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; transition-delay: 0ms !important; scroll-behavior: auto !important; } }

— Editorial print · histórico

Italic como pausa retórica.

La regla "el italic llega después" viene de la tradición tipográfica de El País / Reforma / NYT: la palabra en cursiva del título lleva un beat de respiración. Aquí se traduce a 240ms de delay adicional sobre el último reveal.

— GPU best practices

Solo transform y opacity.

Todas las animaciones se restringen a propiedades aceleradas por GPU: transform, opacity, filter. Cero animación de width, height, top, left. Resultado: 60fps en cualquier laptop de últimos 6 años.

— Apple / Stripe / Linear

Curva .2, .8, .2, 1.

El cubic-bezier(.2,.8,.2,1) es la curva de transición pseudo-default de Apple/Stripe/Linear. Aceleración rápida, desaceleración suave. Lo adoptamos como --ease-out default del sistema.
